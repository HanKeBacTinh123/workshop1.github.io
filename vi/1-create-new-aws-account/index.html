Khởi tạo các thành phần cơ bản

Bước 1:

Ở bước này ta sẽ thực hiện khởi tạo VPC, Subnet, Security Group, Route tables, Internet Gateway, NAT Gateway và EC2 Instance tương ứng.

Khởi tạo VPC:

  1. Đăng nhập vào AWS Console.
  2. Tìm chọn service VPC.
  3. Chọn Your VPCs ở thanh bên trái và chọn Create VPC.

  1. Lần lượt chọn VPC only, điền Name tag VPC A và nhập IPv4 CIDR theo kiến trúc rồi chọn Create VPC. Thực hiện tương tự cho VPC B, C và On Premises. Lưu ý cần bật thiết lập DNS Settings và Network Address Usage metrics settings trong Edit VPC settings.

  1. Xem lại các VPC vừa tạo.

Khởi tạo Subnet tương ứng theo kiến trúc:

  1. Trong giao diện VPC, chọn Subnets, và chọn Create Subnet.

  1. Thực hiện cấu hình các Subnet tương ứng cho VPC A.

  1. Lưu ý cần cho phép tự động cấp phát public ip address cho 2 public subnet.

Các subnet về cơ bản đều giống nhau, thông qua cấu hình route table và cấp phát public IP address mà chúng ta mới phân chia ra Public và Private Subnet.

  1. Xem lại các Subnet vừa tạo.

  1. Thực hiện tương tự cho VPC B, VPC C, On Premises.

Khởi tạo Security Groups tương ứng theo kiến trúc:

  1. Trong giao diện VPC, chọn Security Groups, và chọn Create security group.

  1. Thực hiện cấu hình Security Group cho Public Subnet.
    • Security Group name, nhập Public Subnet SG VPC A.
    • Description, nhập Allow SSH and Ping for servers in public subnet.
    • Chọn VPC A trong mục VPC.
    • Thêm Rules cho phép SSH vào từ MyIP và ping từ Anywhere trong Inbound rules.
    • Kiểm tra Outbound rules và chọn Create security group.

  1. Thực hiện cấu hình Security Group cho Private Subnet.
    • Security Group name, nhập Private Subnet SG VPC A.
    • Description, nhập Allow SSH and Ping for servers in private subnet.
    • Chọn VPC A trong mục VPC.
    • Thêm Rules cho phép SSH vào từ Public Subnet SG VPC A và ping từ Anywhere trong Inbound rules.
    • Kiểm tra Outbound rules và chọn Create security group.

  1. Thực hiện tương tự cho VPC B, VPC C, On Premises.

Khởi tạo Route table tương ứng cho các VPC:

  1. Trong giao diện VPC, chọn Route tables và chọn Create route table.

  1. Thực hiện tạo Public Route Table và Private Route Table cho VPC A rồi gán chúng vào các subnets tương ứng.

  1. Thực hiện tương tự cho VPC B, C.

Khởi tạo Internet Gateway tương ứng theo kiến trúc:

  1. Trong giao diện VPC, chọn Internet Gateways và chọn Create Internet Gateway.

  1. Thực hiện cấu hình Internet Gateway cho các VPC A, B, C và gán vào các VPC đó.

  1. Cập nhật Public Route Table của các VPC.

Khởi tạo NAT Gateway tương ứng theo kiến trúc:

  1. Trong giao diện VPC, chọn NAT Gateways và chọn Create NAT Gateway.

  1. Thực hiện cấu hình Internet Gateway cho các VPC A, B, C và gán vào các VPC đó. Lưu ý chọn Public Subnet theo kiến trúc.

  1. Cập nhật Private Route Table của các VPC.

Khởi tạo EC2 Instances tương ứng theo kiến trúc:

  1. Trên thanh tìm kiếm tìm EC2 và chọn services EC2.
  2. Chọn mục Instances và chọn Launch instances.

  1. Thực hiện cấu hình EC2:
    • AMI là Amazon Linux 2 AMI (HVM).
    • Instance Type t2.micro.
    • Tạo Keypair mới Workshop1.
    • Edit Network Settings theo kiến trúc.
    • Kiểm tra và Launch instance.

  1. Thực hiện tương tự cho các EC2 còn lại. Lưu ý đối với các Private EC2 thì chọn Disable Auto-assign public IP.

  1. Kiểm tra kết nối.
